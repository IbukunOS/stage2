
# REMOTE HUSTLE – STAGE 2 TASK

## DEVOPS & INFRASTRUCTURE – PRACTICAL

## SETUP

Department: DevOps & Infrastructure

## Focus: System Deployment • Configuration • Backup & Security

## Implementation

## 1. Objective

Stage 2 tests your ability to:
● Implement a basic live infrastructure for a startup
● Configure hosting, deployments, and backups
● Apply basic security measures in a real environment
● Produce a system that Remote Hustle staff can actually use

## 2. Core Task

You are required to set up a working infrastructure for Remote Hustle. This is
hands-on and practical.

Your setup must include:

## 1. Hosting & Deployment

● Set up a website hosting environment (FREE shared hosting, VPS, or
cloud )

● Deploy a dummy website or simple app (static HTML or WordPress, or
simple web app)

● Implement a deployment method (FTP, Git push, or auto-deploy if
available)

● Document the deployment process clearly

## Deliverables

● Live site URL or app URL
● Deployment instructions (short guide in Google Doc)

## 2. Backup Implementation

● Set up daily automatic backups of:
○ Website files
○ Database (if applicable)
○ Store backups on a different location (local server, cloud storage,
or Google Drive/Dropbox)

○ Include backup testing: restore at least once to verify

## Deliverables

● Screenshot proof of backup configuration
● Screenshot proof of successful restore

## 3. Security Implementation

● Install SSL certificate (free certificate allowed)
● Create strong password & user roles for access control
● Configure basic firewall or hosting security settings
● Document how someone can safely log in, update, or recover the system

## Deliverables

● Screenshots of SSL, user roles, and firewall settings
● Short write-up of security measures

- Optional Bonus (for extra credit)
● Implement staging environment for testing updates before production
● Automate deployment using GitHub Actions or similar
● Add monitoring/logging for uptime or errors

## 3. Final Submission

Submit a structured document (Google Doc or PDF) including:
● Hosting & Deployment Setup (with URL & instructions)
● Backup Implementation (screenshots & description)
● Security Setup (screenshots & description)
● Optional Bonus Steps (if completed)

File naming format:
Stage2_DevOps_YourID_PracticalSetup.md
